# Inventari Android APK

How to ship the **same mobile web UI** (`MobileApp` / `DynamicMobileApp`) as an installable **Android APK**. The repo today is a Vite + React web app only — there is no `android/` folder or Capacitor config yet. This document is the blueprint for adding one.

For how the mobile shell works in the browser, see [frontend/README-MOBILE.md](../frontend/README-MOBILE.md).

---

## What you get

| In browser (today) | In APK (goal) |
| --- | --- |
| Bottom tabs: Veprime, Transfer, Produkte, Histori, Permbledhje | Same |
| Bottom sheets, card lists, touch targets | Same (WebView) |
| Cookie session login (`credentials: 'include'`) | Same API, with config below |
| Auto mobile UI on Android (`isMobileClient()`) | Yes — Android WebView UA triggers mobile shell |
| Desktop dashboard on wide screens | Hidden on phones; tablet may need `?desktop=1` testing |

The APK is **not a separate React Native app**. It wraps the built `frontend/dist` assets (or loads your hosted URL) inside a native Android WebView via **[Capacitor](https://capacitorjs.com/)** (recommended for Vite projects).

---

## Two packaging strategies

### A — Bundled APK (recommended for “real app”)

1. `npm run build` produces static files in `frontend/dist`.
2. Capacitor copies them into the Android project.
3. The app opens `index.html` locally; API calls go to your **production backend** via `VITE_API_BASE_URL`.

**Pros:** Works offline for shell/assets; fast startup; installable without relying on a static host.  
**Cons:** Each frontend release needs `cap sync` + rebuild APK; must configure CORS and cookies for the WebView origin.

### B — Remote URL shell (fastest to try)

Capacitor WebView loads your live site, e.g. `https://inventari-frontend.onrender.com/?mobile=1`.

**Pros:** No bundled frontend updates; one deploy updates all APK users.  
**Cons:** Requires network; behaves like a dedicated browser; store review may ask why it isn’t a PWA/TWA.

For Inventari, **strategy A** matches “APK just like the mobile version” best.

## Prerequisites

### On your machine

| Tool | Version | Purpose |
| --- | --- | --- |
| Node.js | 20+ | Same as frontend |
| npm | 10+ | Workspace install |
| JDK | **17** (LTS) | Android Gradle builds |
| Android Studio | Latest stable | SDK, emulator, APK/AAB build |
| Android SDK | API **34**+ (target); min **24**+ typical | Capacitor default |

Install Android Studio → SDK Manager → install **Android SDK Platform**, **Build-Tools**, and a **system image** for the emulator.

Set `ANDROID_HOME` (or `ANDROID_SDK_ROOT`) and add `platform-tools` to `PATH` if Gradle cannot find the SDK.

### Hosted backend (required)

The APK cannot use Vite’s dev proxy (`/api` → `localhost:3001`). You need a **public HTTPS API**, e.g. Render backend from [docs/render.md](render.md):

- `https://<your-backend>.onrender.com/api`

Production env on the server:

- `CORS_ORIGIN` — must allow the WebView origin (see [Auth & CORS](#auth--cors))
- `SESSION_SECRET` — min 32 chars
- `SUPABASE_*`, legacy login vars, optional `GOOGLE_CLIENT_ID`

8. **Backend CORS** — allow Capacitor WebView origin(s).
9. **Cookie policy** — verify login session in WebView (see below); adjust `sameSite` if needed.
10. **Google Sign-In** — if used, add Android OAuth client + SHA-1 in Google Cloud (Web client alone is not enough in WebView).

## Step-by-step: Capacitor + bundled APK

Run from the **repository root** unless noted.

### 1. Install Capacitor (in `frontend`)

```bash
npm install -w frontend @capacitor/core @capacitor/cli @capacitor/android
```

### 2. Initialize Capacitor

```bash
cd frontend
npx cap init "Inventari Im" "com.inventari.app" --web-dir=dist
```

Use your real reverse-DNS `appId` (required for Play Store). Example: `com.yourcompany.inventari`.

### 3. Vite production base path

In `frontend/vite.config.ts`, set relative base for Capacitor builds:

```ts
export default defineConfig({
  base: process.env.CAPACITOR ? './' : '/',
  // ...existing config
})
```

Build with `CAPACITOR=1` when syncing to Android.

### 4. Production API URL

Create or extend `frontend/.env.production` (or root `.env.production` — Vite `envDir` is repo root):

```env
VITE_API_BASE_URL=https://YOUR-BACKEND.onrender.com/api
VITE_GOOGLE_CLIENT_ID=your-web-client-id.apps.googleusercontent.com
```

Rebuild whenever the API host changes.

### 5. Build web assets

```bash
npm run build -w frontend
```

### 6. Add Android platform

```bash
cd frontend
npx cap add android
npx cap sync android
```

This creates `frontend/android/` and copies `dist/` into the native project.

### 7. Open in Android Studio

```bash
npx cap open android
```

- **Run** on emulator or USB device (enable Developer options + USB debugging).
- **Build → Build Bundle(s) / APK(s) → Build APK(s)** for a debug APK.
- For Play Store: **Build → Generate Signed Bundle / APK** → **AAB** (recommended) or signed APK.

### 8. Repeat after frontend changes

```bash
CAPACITOR=1 npm run build -w frontend
cd frontend && npx cap sync android
```

Then rebuild in Android Studio.

## Mobile UI behavior in the APK

No separate “APK mode” flag is required today.

| Mechanism | File | APK behavior |
| --- | --- | --- |
| Early `mobile-client` class | `index.html` inline script | Runs on Android UA before React |
| Bootstrap viewport | `lib/mobileBootstrap.ts` | Android viewport width fix |
| Shell selection | `lib/mobileClient.ts` + `App.tsx` | Android UA → `MobileApp` / `DynamicMobileApp` |
| Mobile CSS bundle | `App.tsx` imports `mobile/styles/mobile.css` | Required — do not remove |

Optional hardening for APK-only builds:

- Set `sessionStorage.setItem('inventari-mobile', '1')` once at Capacitor `app` launch so the mobile shell is forced even on large tablets.
- Hide “open desktop site” affordances if you add any later.

## Auth & CORS

The app uses **HTTP-only session cookies** (`inventari_session`, `credentials: 'include'` in `lib/api/http.ts`). Login sets the cookie in `backend/src/auth/session.ts` (`sameSite: 'lax'`, `secure` in production).

### WebView origin

Capacitor Android typically serves bundled files from:

- `https://localhost` (default `androidScheme: 'https'`)

Your backend `CORS_ORIGIN` must include that origin **with credentials**:

```env
# Example — adjust to your Capacitor config
CORS_ORIGIN=https://localhost
```

If you use multiple origins (web + Capacitor), Fastify’s single-string `origin` may need a code change to an array or callback — today `backend/src/app.ts` accepts one `CORS_ORIGIN` string.

### Cookies in cross-origin WebView

- API host: `https://api.example.com`
- App origin: `https://localhost`

The session cookie is stored for the **API host** and should be sent on subsequent `fetch` calls to that host. Test login → `GET /api/session` on a real device early.

If login succeeds but session is lost on the next request:

1. Confirm `secure: true` on cookies (HTTPS API only).
2. Try `sameSite: 'none'` + `secure: true` for the session cookie in production (requires backend change).
3. As a last resort, consider a Capacitor-friendly token header (larger change — not implemented today).

### Google Sign-In

Web flow uses Google Identity Services + `VITE_GOOGLE_CLIENT_ID` (Web client).

Inside Android WebView you may need:

1. **Web client** — authorized JavaScript origins still matter for GIS.
2. **Android OAuth client** in Google Cloud Console with your app’s **package name** and **SHA-1** (debug keystore for dev; upload keystore for release).

Without Android client configuration, **Hyr** with Emri/password still works; Google button may fail in the APK until configured.

## Testing checklist

Before distributing an APK:

- [ ] Cold start → login screen renders (mobile layout, not desktop table).
- [ ] **Hyr** with legacy or dynamic account → dashboard tabs appear.
- [ ] Session persists after killing and reopening the app.
- [ ] **Veprime** — add product row, finalize (bottom sheet confirm).
- [ ] **Transfer** — Nga/Te, date, finalize.
- [ ] **Histori** — filters, advanced sheet, **Shkarko** export download (may open system download UI).
- [ ] **Permbledhje** — date range, Excel export.
- [ ] Dynamic account — location picker sheets, N-column stock on Produkte cards.
- [ ] Onboarding wizard (`/onboarding`) on new dynamic signup.
- [ ] Back button — does not exit app unexpectedly on root tab (may need `@capacitor/app` back-button listener later).
- [ ] Keyboard — Ora/time sheets and numeric fields usable on small screens.

## What stays the same vs browser mobile

| Area | Change for APK? |
| --- | --- |
| React components (`src/mobile/*`, `dynamic/mobile/*`) | No |
| Business rules, API payloads | No |
| Backend Supabase / migrations | No |
| Desktop UI (`DashboardPage`) | Not shipped as primary UX on phones |
| `VITE_API_BASE_URL=/api` relative path | **Yes** — must be absolute production URL |
| CORS / cookies | **Yes** — configure for WebView |
| New native features (push, biometrics, camera) | Optional later via Capacitor plugins |
